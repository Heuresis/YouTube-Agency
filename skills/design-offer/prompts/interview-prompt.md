# Interview Prompt — Design Offer

When `/design-offer` is invoked but offer compartment is empty, run interview-mode in this order.

## Cluster 1 — Existing offer state
1. Do you currently have a paid offer? Name, price, type, current customer count, current results.
2. If yes: what % of revenue does this offer produce? What's working? What's not?
3. If no: what offer are you planning? Why this offer specifically?

## Cluster 2 — Avatar fit
4. Confirm the avatar from `/build-audience`. Is the offer for this avatar?
5. Confirm dominant limiting belief and required transformation type.
6. What's the audience's current buying-power band?

## Cluster 3 — Transformation
7. Specific before-state your buyer is in?
8. Specific after-state in 90/180/365 days?
9. What mechanism gets them there? (named, ownable, 2-4 words)

## Cluster 4 — Mechanism
10. What's the 1-2 key insights you bring that competitors miss?
11. What sequence-reversal, reframing, or methodology defines your approach?
12. What's the named mechanism (working title)?

## Cluster 5 — Value stack
13. What deliverables does the program include?
14. What's the perceived value of each? (USD)
15. Which objection from the audience library does each deliverable address?

## Cluster 6 — Bonuses
16. What 2-4 bonuses can you add that address tail-end objections?
17. Each bonus value + objection addressed?

## Cluster 7 — Qualification
18. Who is this offer specifically for? (3-5 must-haves)
19. Who is it NOT for? (3-5 must-not-haves)
20. How will you filter? (application / call / typeform)

## Cluster 8 — Guarantee
21. What's the audience's dominant fear?
22. What guarantee structure absorbs that fear?
23. Specific terms?

## Cluster 9 — Ladder
24. What's the lead magnet? (free)
25. Tripwire? (optional)
26. Upsell? Continuity?
27. High-ticket above this offer?

## Cluster 10 — Economics
28. Estimate LTV: revenue per customer × continuity attach × continuity term
29. Estimate CAC: production cost amortized + sales ops + ads if any
30. LTV:CAC ratio? Below 3:1 = revise.

## Synthesis
Run the document per SKILL.md. Validate economics. If LTV:CAC < 3:1, return with diagnosis + revision options (raise price / add continuity / reduce CAC).
