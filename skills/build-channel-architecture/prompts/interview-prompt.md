# Interview Prompt — Build Channel Architecture

## Cluster 1 — Channel promise
1. In one sentence: who is this channel for, what does it teach them, what outcome does it produce?
2. Does the promise pass the "viewer can repeat after 3 videos" test?
3. What does the channel do that no other channel does?

## Cluster 2 — Pillars
4. What 3-5 content pillars will the channel run?
5. What % volume per pillar? (Default 50/25/15/10)
6. Each pillar's mode (authority / reach / hybrid)?
7. Each pillar's conversion bridge to the offer?

## Cluster 3 — Foundational playlist
8. What 5-10 videos should a new subscriber watch first to be ready for the offer?
9. What belief does each video install?
10. What awareness-level transition does each enable?

## Cluster 4 — Belief-installation series
11. From the offer's 8 beliefs, what video installs each?
12. What hook formula matches each belief's emotional terrain?

## Cluster 5 — Handoff to offer
13. End-screen pattern (which links?)
14. Description CTA stack (ordered)?
15. Pinned comment pattern?
16. Email sequence cross-link?
17. Community tab cadence?
18. Where does the pinned VSL appear?

## Cluster 6 — Multi-awareness
19. % of audience at each awareness level?
20. Role for each level's videos?
21. Bridge between adjacent levels (a specific bridge video)?

## Cluster 7 — Positioning
22. Position in market (against archetypes)?
23. 3-5 archetypes the channel resembles?
24. 3-5 archetypes the channel explicitly is not?
25. Anti-positioning statement (1 paragraph for About page)?

## Synthesis
Assemble Channel Architecture Document. Verify 8-belief mapping, awareness bridges, handoff path completeness.
