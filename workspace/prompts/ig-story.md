# ig-story — Stories sequence generator

> Produces a film-ready **7-story sequence** for Instagram — the warm-audience surface of the attention engine (`workspace/STRATEGY.md`). Stories do two jobs: **nurture the warm audience** (de-risk the creator the way the YouTube proof channel does) and, in a launch window, **drive registrations / bookings** (attention → link sticker / bio → the destination). Match the `write-script` voice bar; the channel's canonical spoken-pass draft (named in `workspace/CALIBRATION.md`) is the canon. Proof first, ask second.

## Before you generate — read these
1. `workspace/VOICE.md` — the spoken-voice rules + reserved-vocabulary discipline + banned vocab. **Non-negotiable.** (Overlays are tiny, so every word is load-bearing — voice rules matter *more* here, not less.) Gate: `spec/voice-gates.md`.
2. `workspace/PROOF-BANK.md` — every number on screen traces to a row here. **Never invent a number** (INV-6). If it isn't `[VERIFIED]`, it's a `[CONFIRM: …]` slot — not a number you make up.
3. `workspace/STRATEGY.md` — the funnel, the Stories charter, any open launch window. Stories serve the funnel; they are not vibes-only.

## Inputs (ask for any that are missing — don't guess)
- **Intent:** `A` nurture/proof · `B` drive-to-launch (launch window) · `C` build-in-public day-in-the-life. (Defined below. Default `A`.)
- **The proof / the moment:** which PROOF-BANK row(s) or which real thing-that-happened-today is this built on? If the core number isn't `[VERIFIED]`, mark `[CONFIRM: $X]` on that story.
- **The frame** (pick 1 of the channel's recurring frames, `workspace/VOICE.md`).
- **The disqualifier:** operator-fit (default for B) or client-fit. See the disqualifier section.
- **CTA destination:** DM keyword / link to a YT case study / pinned VSL (A) · launch registration link + countdown (B) · poll/question + soft bridge (C).
- **Launch context:** is today inside a launch window (`workspace/STRATEGY.md`)? If yes, intent skews `B` and every sequence ends on the launch destination.

## The 7-story spine (universal — the backbone of all three intents)

Each story = **one tap**. Format for every story: **overlay ≤10 words + visual description + one interactive element.** Open the loop on Story 1, pay it off by Story 5, earn the ask on Story 7.

| # | Beat | Job | Default interactive |
|---|---|---|---|
| **1** | **The hook** | One line over a real artifact (screenshot / dashboard / a number / a late-night moment). Open the loop. | Quiz or poll (a low-friction tap that *is* the loop) |
| **2** | **The problem** | Show the broken thing. The leak, named. | Poll — "happen to you? / yep / no idea" |
| **3** | **The cost** | The bleed in numbers. workflow × frequency × cost. | Quiz — "what's that costing? A/B/C" |
| **4** | **The fix** | What we built. Screen-capture snippet / before-after / the stack. | Reaction or question — "want to see it?" |
| **5** | **The proof** | The payoff artifact. The dashboard, the result, the receipt. | Question — "what would you automate first?" |
| **6** | **The disqualifier** | Filter for fit. The authority move (below). | **Poll = the qualifier** — "sound like you? / yes / not me" |
| **7** | **The CTA** | Imperative + image, never a question. One ask. | Link sticker + DM-trigger (and countdown in launch window) |

**Overlay rules (the tiny-canvas version of the voice):**
- ≤10 words. Fragments win. One idea per story.
- **Numerals are fine on the overlay** (`{{$X}}`, `{{N}} clients`) — it's read silently and scannability rules. The **full-number read** applies only to **voiceover**, if the creator is talking over the story.
- No BUT/THEREFORE connectors. Contrast lives across two stories, not inside one line.
- "And"-openers, one-word punchlines ("Gone." "Receipts.") — yes.
- **Show, don't claim** — the overlay names the claim, the *visual* proves it. No overlay ships without an artifact behind it.
- **Default no emoji.** One dry/self-deprecating emoji max per sequence, only if it's genuinely the creator's register. Never hype emoji. Stickers are native UI, not emoji-in-text.
- Banned on sight: "Imagine if…", "What if I told you…", "secret", "magic", "passive income", "set-and-forget" (except when *rejecting* it in a disqualifier), any `spec/BANNED-VOCABULARY.md` term.

## The 3 sequence intents (how the spine flexes)

| Intent | Audience state | Spine emphasis | Story 7 CTA | Frame default |
|---|---|---|---|---|
| **A — Nurture / proof** | Warm follower researching "are they legit?" | Full 7 beats, proof-heavy (4–5 carry the weight). | **DM keyword** → a breakdown/walkthrough · or **link** to the YT case study / pinned VSL. Soft. | the channel's fix-over-tech frame |
| **B — Drive-to-launch** | Saw an ad/reel, deciding whether to register | Proof 1–5 compressed, then the turn: "I'm breaking down the whole system [day]." | **Link sticker → registration + Countdown sticker** (real start time). DM keyword backup. | mechanism over outcome |
| **C — Build-in-public day-in-the-life** | Casual / parasocial crowd | Cold-open *moment*, honest grind, a real artifact, the honest cut. Light sell. | **Poll / question** for engagement + soft bridge ("full story on YouTube", "link in bio"). | build-in-public honesty |

**Intent C beat swap** (reach register — unguarded beats rehearsed):
S1 cold-open moment (the beat *before* the realization) · S2–3 what's actually happening today (a win **and** a miss — keep the miss) · S4–5 the artifact (closed-deal screen, build-in-progress, a client DM) · S6 the **honest cut / anti-guru** beat ("everyone says this is easy — it's more than it seems") · S7 question sticker + soft bridge. Powerful, but route it: every reach sequence bridges to a proof story or the bio.

**Cadence (launch window):** post the 7 as one **burst** (back-to-back, so it plays as a sequence). During a launch window run one sequence/day, alternating B with a single A proof-drop, so the feed isn't all-ask.

## The disqualifier-as-authority move (Story 6 — do not skip it)

Filtering *is* the authority signal. The person who says "this isn't for everyone" reads as someone with something worth qualifying for. On a story the **poll becomes the qualification mechanic** — the viewer self-selects with a tap, and the "yes" tappers are your warmest Story-7 audience.

Two flavors — pick by who the sequence is selling to:

- **Operator-fit (default for B → the program/offer):** anchored on the beliefs the creator will defend (`workspace/VOICE.md` recurring frames).
  - "If you won't actually learn the system — skip this."
  - "Want passive income? This isn't it." *(rejecting the banned term = on-brand)*
  - Poll: **"Building the real thing? / Yes / Just looking."**
- **Client-fit (when positioning the service offer):** the revenue/stage filter from `workspace/STRATEGY.md`.
  - "Under {{revenue floor}}? The math doesn't work yet."
  - Poll: **"Sound like your business? / Yep / Not yet."**

Never soften it into a question-close. State it flat, let the poll do the filtering.

## Sticker + DM-trigger mechanics (the conversion layer)

| Sticker | What it does | Use on |
|---|---|---|
| **Poll** (2 options) | Lowest-friction tap; algorithm signal; self-selection | S1 hook, S2 "happen to you?", **S6 the qualifier** |
| **Quiz** (multi + right answer) | Teaches the cost math; opens the loop | S1 ("where's the leak?"), S3 (the cost) |
| **Question** (open box) | Mirrors the live-diagnosis format; high-intent replies → DM threads | S5, S7 (A/C) |
| **Link** | The conversion mechanic | **S7** → registration / pinned VSL / YT case study |
| **Countdown** | Reminder *without* false scarcity | **S7 in launch window** → set to the real start time |
| **Emoji slider / reaction** | Light engagement only | S4 ("how clean is that?") |

**DM-trigger mechanics** — one word, ALL CAPS, in quotes. Replies fire a DM thread (warm-lead capture) *and* spike reach. Keep keywords stable so they compound. Define the channel's keyword set in `workspace/STRATEGY.md` (e.g. `{{AUDIT}}` → a walkthrough of their stack · `{{SYSTEM}}` → the build/case-study breakdown · `{{VSL}}` → the pinned VSL · a launch keyword during windows).

**Link / CTA discipline (`workspace/VOICE.md` CTA architecture):**
- Every link carries context — never a bare "link here / click below." Pair it with the image and the reason ("Tuesday. Free. Tap to save your seat.").
- **Countdown sticker = allowed** (it's a reminder to a real start time). **"Doors close at midnight" fake scarcity = banned.** Different things.
- Banned CTAs: "like and follow" as the *only* ask · "free strategy call" with no deliverable · question-mark closes ("what do you think?") · "click my bio" with no context (INV-17).
- One ask per sequence. Earn it: Story 7 only lands because Stories 4–5 paid off.

## The procedure
1. **Pick intent + the proof + one frame.** One sequence, one job.
2. **Map the 7 beats** to the spine (flex per intent table). Decide the loop: what Story 1 opens, Story 5 closes.
3. **Write overlays spoken-faithful** — ≤10 words, fragments, no BUT/THEREFORE, one-word punchlines. Read each aloud; if your mouth feels weird, the line's wrong.
4. **Spec the visual for every story** — the artifact that *proves* the overlay (payment/booking screen, screen-capture, board, dashboard, the DM, the analytics screen). No naked claims.
5. **Place one interactive per story**, weighted to the beat (poll early + on the disqualifier; question on proof; link/countdown on S7).
6. **Trace every number** to a PROOF-BANK row; open a `[CONFIRM: …]` slot for anything not `[VERIFIED]`.
7. **Voice-check** (`spec/voice-gates.md`) + signal-check (`workspace/prompts/signal-check.md`, target ≥7).

## Hard constraints (the sequence fails review if any break)
- **No invented numbers.** Every figure → a `[VERIFIED]` row or a `[CONFIRM: …]` slot.
- **Every overlay ≤10 words; every story has an artifact** behind it (show, don't claim).
- **Voice rules applied** to overlays + any voiceover; read-aloud clean.
- **Reserved-vocabulary discipline** — simple proof-first language only on IG.
- **Story 6 disqualifier present** — the authority move is not optional.
- **Earn the pitch** — no link/ask before the proof (S4–5) lands.
- **Income claims** carry the disclosure frame (`workspace/PROOF-BANK.md` honesty section); client ROI ≠ audience income; never guaranteed (NEVER-11).
- **One destination.** Every sequence ties off to exactly one CTA.

## Worked example skeleton — Intent A (fill from your own bank)
*Frame: the fix-over-tech frame. Built on {{PROOF-BANK row}} `[VERIFIED]`. Loop: S1 "where's the leak?" → S5 the live dashboard.*

1. **Hook** — Overlay: "A {{niche}} was losing {{$X}} every month." Visual: the booking/quote screen with the dead hours visible. Quiz: "Where's the leak? A/B/C."
2. **Problem** — Overlay: "Every {{request}} after {{time}} went to {{dead end}}." Visual: the unread queue. Poll.
3. **Cost** — Overlay: "That's {{N}}% of new {{bookings}}. Gone." Visual: the plain math, handwritten, not slick. Quiz.
4. **Fix** — Overlay: "We built {{the fix}} in {{timeframe}}." Visual: screen-capture of the flow running end to end. Reaction.
5. **Proof** — Overlay: "Now the owner sees every {{booking}}, live." Visual: the dashboard. Question.
6. **Disqualifier** — Overlay: "If you won't learn the system — skip this." Visual: creator to camera, plain, no edit. Poll qualifier.
7. **CTA** — Overlay: "DM '{{KEYWORD}}' — I'll map yours." Visual: the diagnostic-lens graphic. DM-trigger + link sticker → the full YT case study / pinned VSL.

**Proof trace:** every `{{…}}` above resolves to a PROOF-BANK row + tag before this ships.

## Pre-post validation (run before it ships)
1. **Voice** — could every overlay come out of the creator's mouth on a live? Generic-creator line → sharpen with a named number / real tool / contrarian belief.
2. **Reserved-vocab lint** — proof-first language only; no program vocabulary.
3. **Proof path** — every number traces to a row; `[CONFIRM: …]` slots flagged; disclosure attached if any audience-income claim.
4. **Structure** — 7 stories, ≤10-word overlays, artifact behind each, disqualifier present, one CTA, earned after proof.
5. **Signal ≥7** — recast 4–6, reject 0–3.
