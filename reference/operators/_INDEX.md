## Common stacking patterns

These archetypes are rarely solo. Most channels layer two — one for authority, one for reach.

| Channel pattern | Primary archetype | Secondary archetype | Why this stack works |
|---|---|---|---|
| High-ticket B2B coach | long-form-vsl-specialist | contrarian-narrator | The VSL specialist sells; the contrarian narrator pulls reach the VSL needs to feed |
| Tech explainer with offer | analytical-explainer | mechanism-first-educator | Density wins authority; mechanism wins durability |
| Personal-finance creator | personal-finance-mentor-archetype | story-driven-creator | Credibility gates; narrative converts |

| Long-form-VSL with broad appeal | long-form-vsl-specialist | contrast-format-viral-specialist | Authority converts; reach fills the funnel |

## Selection logic

Use this rough decision flow when calibrating a channel:

1. **Is the back-end offer high-ticket ($5K+) and application-funnel-driven?**
   - Yes, niche is agency-acquisition → `agency-acquisition-operator-archetype` primary
   - Yes, niche is general high-ticket → `long-form-vsl-specialist` primary
2. **Is the niche personal finance or wealth-building?**
   - Yes → `personal-finance-mentor-archetype` primary (credibility-gated)
3. **Is the niche tech, data, or system-explaining?**
   - Yes → `analytical-explainer` primary
4. **Does the operator have a named mechanism worth owning?**
   - Yes → `mechanism-first-educator` primary or layered onto another archetype
5. **Is the niche productivity / knowledge-work / tool-driven?**
   - Yes → `productivity-systems-builder` primary
6. **Is the niche mindset, philosophy, or identity-transformation?**
   - Yes → `mindset-philosophical-creator` primary
7. **Is the niche creator-economy / about-creators?**
   - Yes → `creator-economy-meta-creator` primary
8. **Does the operator have ongoing client cases worth documenting?**
   - Yes → `case-study-archivist` primary or layered
9. **Does the channel need reach-mode growth?**
   - Yes → layer `contrast-format-viral-specialist` as secondary
10. **Does the operator have a strong contrarian frame against the niche's consensus?**
    - Yes → layer `contrarian-narrator` as secondary or primary
11. **Does the operator have lived narrative continuity worth carrying across videos?**
    - Yes → layer `story-driven-creator` as secondary

## How to use this catalog

Each archetype file documents:
- **Frontmatter** — slug, descriptor, niche fits, mode strength, signature frameworks, applicable when conditions
- **Thesis** — the worldview that shapes the operator's work (3–5 paragraphs)
- **Signature Frameworks** — the structural patterns the operator is known for (5–8 frameworks)
- **Distinctive Moves** — specific tacit behaviors that differentiate this archetype (8–12 moves)
- **Lineage** — the structural traditions this archetype descends from
- **When to Invoke** — the channel/creator conditions where this archetype's playbook applies + when to use a different archetype instead

Read the relevant archetype file fully when calibrating a new channel, designing a back-end offer, or auditing a channel's positioning. Reference the file when running `/build-niche`, `/build-channel-architecture`, `/extract-creator-voice`, `/design-offer`, and `/series-architect`.

## Anti-watchlist

These archetypes are anonymized, structural patterns. Do not attribute archetypes to specific real operators in the workspace. The descriptors ("the long-form-VSL specialist," "the analytical-explainer") are the canonical references. Real operator names, real channel names, real brand names are forbidden in archetype documentation, in skill outputs, and in any artifact that ships.
