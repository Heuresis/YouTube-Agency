# Script-Section Specimen 016 — Architecture-walkthrough body section

## Structure (paraphrased)
- Re-hook (10–15s): "OK — let's get into the architecture itself. This is where most people get this wrong."
- Diagram introduction (30–60s): Visual on screen. Speaker walks through the components left-to-right or top-to-bottom. Single sentence per component.
- Trade-off discussion (90–120s): Each architectural choice's trade-off, named explicitly. "We chose [option A] over [option B] because of [specific trade-off]. The cost is [specific cost]."
- Failure-mode tour (60–90s): Walk through the 2–3 most common ways this architecture breaks. "If you skip [component], you'll see [specific failure]."
- Decision rule (30–45s): Provide the heuristic. "Use this architecture when [specific condition]. Don't use it when [specific condition]."
- Transition (10–15s): "Now that we have the architecture, the next decision is [next section]."

## Why this works (the imitation lesson — 2–4 sentences)
Architecture-walkthrough sections earn senior-engineer retention by walking through trade-offs explicitly — that's the cognitive layer the audience cares about. The "single sentence per component" discipline keeps the diagram-walkthrough from becoming a slog. The decision-rule heuristic at the end (Beat 5) is the load-bearing element; it converts the section from explainer to applied tool.
