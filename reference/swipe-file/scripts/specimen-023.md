# Script-Section Specimen 023 — Framework-component body section

## Structure (paraphrased)
- Re-hook (10–15s): "OK — component [N] of the framework. This is the one that does the heavy lifting."
- Component definition (30–45s): Name and define. "Component [N] is [name]. Its job is [specific function]."
- Mechanism (60–90s): Why this component works. "It works because [structural principle]."
- Worked example (90–120s): Show the component in action. "When I apply this to [example], here's what it looks like..."
- Without-this-component contrast (45–60s): Show what fails when this component is missing. "Without it, you get [specific failure mode]."
- Tactical instruction (45–60s): How to install this component in the audience's context. "To put this in place, you need to [specific actions]."
- Transition (10–15s): "OK — [N] components down, [M] to go. Next is..."

## Why this works (the imitation lesson — 2–4 sentences)
Framework-component sections work because each component is structurally complete — definition, mechanism, example, contrast, instruction — which means the audience can extract value from any single section. The contrast beat (Beat 5) is the load-bearing element; showing what fails without the component is what makes its necessity visceral rather than abstract. The "next is..." transition (Beat 7) maintains framework momentum.
