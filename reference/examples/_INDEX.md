# Examples

> Filled end-to-end worked examples — a complete pass through the system for one published video, preserved as a study artifact. This directory ships empty in the template and fills as the channel operates; `/library-compound` files a cycle here when a video's full artifact chain is worth keeping.

## What belongs here

| Artifact | What it is | When to file it |
|---|---|---|
| `<video-slug>/` | One directory per worked example: research brief → script → thumbnail + editor briefs → publish checklist → retention postmortem, for a single published video | Top-decile performance, a clean failure worth studying, or the first instance of a new format |

## Filing rules

- One directory per video, named by slug. Keep the artifacts as they shipped — worked examples are evidence, not templates; do not retro-edit them to look better.
- Strip anything that can't ship in a template: client names, revenue not already public, credentials.
- Until this directory has entries, `templates/example-scripts.md` and the `swipe-file/` specimens are the reference points for what good looks like.
