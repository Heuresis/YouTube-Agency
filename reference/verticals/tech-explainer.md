# Tech Explainer — Vertical Playbook

> Vertical-specific playbook for tech-explainer YouTube channels (programming, AI / ML, dev tools, B2B-software analysis, infrastructure deep-dives). Read alongside `reference/benchmarks/tech-explainer.md`.

## Vertical positioning advice

Tech explainer is a depth-driven, evergreen-leaning, technical-audience niche. Positioning gates:

1. **Genuine technical depth.** The audience can detect surface-level content within minutes. The operator needs lived implementation experience or deep research-level understanding of the domain.

2. **A specific technology or domain expertise.** Generic "tech channel" is dead on arrival. Specific positioning ("AI/ML for backend engineers," "Postgres internals," "DevOps for Series B startups") is the moat.

3. **A clear teaching style — analytical-explainer or mechanism-first.** The vertical's two dominant archetypes. Analytical-explainer wins on density; mechanism-first wins on framework ownership.

4. **Source citation and verifiability.** Audiences expect on-screen citations, links to underlying data, code repositories, documentation. The operator's credibility is built on verifiability.

5. **Update discipline.** Tech moves fast. Stale tutorials become liabilities. The operator's update cadence is operational discipline.

## Typical offer structures

- **Technical consulting / advisory** — $200–$2,000+ per hour or project-based
- **Specialized cohort courses** — $497–$5,000; multi-week technical programs
- **Self-paced courses** — $97–$2,000; technical-skill-acquisition
- **B2B services / agency arm** — $5K–$100K+ project-based
- **Developer tools / SaaS products** — $20–$200/mo; product-led growth from the channel
- **Books / paid newsletters** — $30 / $5–$30/mo; ancillary

OS bullseye: creators with consulting / specialized cohort offers in the $1K–$10K range, or B2B-services arms.

## Typical audience awareness distribution

- Unaware: 5–10%
- Problem-aware: 20–25%
- Solution-aware: 30–35%
- Product-aware: 25–30%
- Most-aware: 10–15%

Tech audiences skew more product-aware than general audiences because they actively evaluate technologies and have prior context.

## Channel architecture archetypes that work

### The analytical-explainer architect

Library organized as a knowledge-graph. Foundational playlist establishes the technical foundations. Pillar playlists per domain (e.g., backend architecture, distributed systems, DevOps, AI / ML, frontend). Within each pillar, depth-tiered content (intro → intermediate → advanced).

### The mechanism-first-educator architect

Channel owns a specific technical method or framework. Foundational playlist introduces the framework. Pillar playlists deepen specific aspects. Library compounds as a method-specific curriculum.

### The productivity-systems-builder architect (in tech)

Channel demonstrates the operator's actual technical workflow. Build-in-public content. Tool-stack curriculum. Productivity-systems-builder archetype applied to technical work.

## Content pillars typical for this niche

Recommended 4–6 pillars:

1. **Foundations / fundamentals** (15–20%) — entry-level domain education
2. **Deep-dive technical explainers** (25–30%) — analytical-explainer signature content
3. **Tool / framework reviews** (15–20%) — comparison and review
4. **Case studies / build-in-public** (15–20%) — applied demonstration
5. **Industry / market analysis** (10–15%) — timely commentary on tech trends
6. **Tutorials** (10–15%) — instructional / how-to

## Voice of viewer in this niche

Recurring pain language:

- "I understand the basics but I don't know how to [advanced topic]"
- "Tutorials all stop at [the easy part]"
- "I can't find good content on [specific technology]"
- "I'm a [role] but I don't understand [adjacent domain]"
- "I'm [stuck on / confused about] [specific technical concept]"
- "I keep seeing [hyped technology] mentioned — what is it actually?"

## Conversion path specific to tech

Pinned VSL conversion:

- Length: 25–45 minutes (tech audiences accept long-form when it's dense)
- Hook: technical-curiosity or contrarian-tech-take
- Hero story: technical-credibility (years building / shipping / running)
- Mechanism explanation: deep; not consumer-friendly; the audience tolerates and prefers depth
- Price anchoring: ROI-on-technical-skill or career-tier-investment framing
- Crossroads close: career-trajectory comparison; specific to tech career stages

## Common failure modes specific to this niche

### Surface-level depth

**Failure**: the operator covers too many topics shallowly. Audience finds deeper content elsewhere; subscriber retention is poor.
**Prevention**: depth-over-breadth discipline. Better to have 30 deep videos than 100 shallow ones.

### Stale tutorials

**Failure**: tutorial videos using old tool versions, deprecated APIs, outdated frameworks. The audience gets misled and the operator's credibility erodes.
**Prevention**: update discipline. Annual library audit. Update notes on outdated content. Re-record canonical tutorials when major versions ship.

### Source-citation neglect

**Failure**: making technical claims without sources. Technical audiences fact-check; uncited claims lose credibility.
**Prevention**: on-screen citations. Description links to sources. Code repositories where applicable.

### Over-explanation

**Failure**: explaining basic concepts the audience already knows. Patronizing; high drop rate.
**Prevention**: audience-tier targeting. Don't explain functions to senior engineers. Don't explain advanced patterns to bootcamp grads.

### Under-explanation

**Failure**: assuming prior knowledge the audience doesn't have. Confusing; high drop rate.
**Prevention**: scope discipline. State the prerequisites at the start. "This video assumes you understand X."

## Regulatory considerations

Generally low regulatory burden:

- **FTC**: #ad disclosure on sponsored / affiliate content
- **Earnings claims** (for tech-career-coaching content): substantiated only
- **AI / ML claims**: as the field's regulatory environment evolves (2025–2026), evolving compliance considerations may emerge specific to AI products
- **Open-source licensing**: respect attribution / license terms when demonstrating open-source code

## Library compound entry priorities

- Hook patterns by sub-domain
- Audience-tier markers (which content works for which level)
- Update flags (which videos are due for re-record)
- Cross-video reference graph (which videos cite which)
- Search-traffic-strong topics for evergreen-library deepening

## Cross-vertical dependencies

Tech explainer intersects with:

- **Productivity / career** (developer-productivity, knowledge-management for engineers)
- **Business education** (tech-business-strategy, B2B-SaaS-operations)
- **Creator economy** (technical-content creator operations)

The vertical typically does not borrow heavily from finance / mindset / fitness verticals.

## Channel-stage progression

Tech-explainer channels typically progress through:

- **Months 0–6**: niche-finding; technical-credibility-building; library 5–25 videos
- **Months 6–18**: domain-deepening; library 25–75 videos; first conversion infrastructure
- **Months 18–36**: library compounds; long-tail traffic builds; consulting / cohort offers monetize
- **Years 3+**: domain-authority status; library at 100+ videos; long-tail revenue dominant

The library compound is the dominant lever. Tech videos are evergreen-strong; a well-built tutorial earns views for 3–5+ years.
