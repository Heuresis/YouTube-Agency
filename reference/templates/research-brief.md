# Research Brief Template

> Per-video research pack template. The research brief is built before scripting, contains the source material the script will draw from, and is the input to `/write-authority-script` or `/write-reach-script`. Quality of the research brief sets the ceiling on the quality of the script.

## Frontmatter

```yaml
video_title: <working title>
mode: <authority | reach | hybrid>
awareness_target: <unaware | problem-aware | solution-aware | product-aware | most-aware>
content_pillar: <pillar from compartment 5>
research_hours_estimated: <integer>
researcher: <name or "operator">
script_writer: <name or "operator">
deadline: <date>
```

## Section 1 — Topic framing

### The thesis

```
The thesis of this video: <1–2 sentences>
```

This is what the video argues, demonstrates, or teaches. Specific. Not a topic ("YouTube growth"); a thesis ("the algorithm rewards CTR x AVD x session-continuity, in that order, and most operators optimize the wrong one").

### The audience target

```
Primary audience: <segment + scale + niche>
Awareness level: <one of 5>
Pain language: <3–5 verbatim phrases from voice-of-viewer>
Limiting belief: <worthless | helpless | hopeless — see compartment 2>
Sophistication stage: <naive | aware | skeptical | exhausted>
```

### The promise

```
What the viewer will leave with: <specific, measurable transformation or model>
```

The promise is what the title-thumbnail-unit (and the hook) will deliver. INV-10 alignment starts here.

## Section 2 — Source material

### Primary sources

List 3–7 primary sources the script will draw from:

```
1. <Source name / URL>
   - Why this source: <1 sentence>
   - Key claims to use: <bulleted>
   - Quotes to potentially cite: <bulleted with attribution>

2. ...
```

Sources can be:
- Industry reports / published data
- The operator's own historical content (referenced; not duplicated)
- Voice-of-viewer captures (anonymized phrases from comments / DMs)
- Competitor / adjacent-archetype content (for context only — never imitate verbatim)
- Original interviews / surveys conducted by the operator
- The operator's own lived experience (logged; specific moments)

### Secondary sources

For supplementary context:

```
1. <Source name / URL>
   - Use case: <1 sentence>
```

### What sources to avoid

- Unverified claims (pass the 30-second screenshot test — INV-5)
- Fabricated stats (INV-6)
- Sources from operators or channels on the watchlist (real operator names; the OS uses anonymized archetype descriptors instead)
- Opinion-as-fact (clearly distinguish opinion from cited research)

## Section 3 — Frameworks to apply

Which frameworks from `reference/frameworks/` apply:

```
Primary framework: <e.g., 8 required beliefs, BUT-THEREFORE engine, contrast format, mechanism-first explanation>
Secondary framework: <e.g., loop architecture, three-brain decision sequence, awareness ladder>
```

The framework choice determines the script's structural spine.

## Section 4 — Hook candidates

Generate 5–10 hook variants per the channel's hook formulas. Tier them.

```
Hook 1 (primary): <hook text>
- Formula: <PAST | AID | Contrast | Statistic | Curiosity-Gap | Bound-Promise | Contrarian-Frame | Social-Proof-Stack>
- Voice-of-viewer phrase used: <yes/no; which>
- Predicted retention: <high | medium | low>

Hook 2 (alternate): <hook text>
- ...

(continue for 5–10 hooks)
```

The hook bank from compartment 6 informs this section. New hooks are added to the bank for cycle-2.

```
Title 1 (primary pillar): <title>

- Predicted CTR: <high | medium | low>

- ...

(continue for 5–10 titles)
```

## Section 6 — Thumbnail concepts

## Section 7 — Examples / case studies / data

Specific examples the script will use. Each anchored:

```
Example 1: <name / segment / situation>
- Source: <where this came from>
- Verification: <how it passes the 30-second screenshot test>
- Anonymization: <how the example is anonymized — segment + scale + niche, never names>
- Where in the script: <section>

Example 2: <name / segment / situation>
- ...
```

For data (statistics, benchmarks):

```
Data point 1: <claim with number>
- Source: <URL or document>
- Date / freshness: <when this data was published>
- Caveats: <any limitations to acknowledge>
- Where in the script: <section>
```

For authority videos: list the audience's likely objections and how the script addresses each.

```

- Section: <where in the script>

- Response: <how the script handles this>

```

## Section 10 — Verification

The brief's claims must pass:

- [ ] All quoted statistics have a source
- [ ] All case studies are properly anonymized
- [ ] All claims pass the 30-second screenshot test (INV-5)
- [ ] No fabrication (INV-6)
- [ ] No real operator names (anti-watchlist)
- [ ] No banned vocabulary (INV-7)
- [ ] Voice-of-viewer phrases captured from compartment 2

## Research-brief deliverables

- The completed brief document (this template, filled in)
- Source files / links / archives where the script-writer needs them
- Voice-of-viewer phrase collection (extracted from compartments)
- Visual reference if applicable (case-study screenshots, framework diagrams, etc.)

## Quality bar

The research brief is the input to the script. Bad brief → bad script. Quality markers:

- **Specificity**: every claim is anchored to a source, an example, or a voice-of-viewer phrase
- **Verifiability**: claims pass the truth gate
- **Voice**: voice-of-viewer language is captured, not paraphrased
- **Frame-fit**: the framework chosen genuinely fits the topic; not jammed onto the wrong topic
- **Audience-fit**: the awareness level and sophistication stage match what the audience actually is
