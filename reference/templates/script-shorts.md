# Shorts Script Template

> 30–60 second Shorts script template. Used for vertical-format videos in the Shorts feed. Mode: typically reach (occasionally authority for tightly-scoped product-aware content). Target length: 30–60 seconds (75–150 words). Hook formula: contrast / contrarian / story-beat / statistic-shock compressed.

## Section 0 — The 3-second hook (0:00–0:03)

The Reptilian-brain decision happens in 0–1 second; the verbal-promise commitment in 0–3 seconds. INV-11 for Shorts: minimum 2 mechanisms in first 3s + 1 in 3–15s.

```
[VISUAL: <strong, motion-rich, on-screen text dominant>]
[AUDIO: <hook line — short, declarative>]

<HOOK SENTENCE 1 — 5–10 words; activates curiosity, contrast, or contradiction>
```

The body's premise. The Limbic engagement. Voice-of-viewer language tight.


The lesson, contrast-resolution, or story-beat payoff. Where the value-add lives.

```

```

## Production cues for Shorts

- **Format**: vertical 9:16; center-of-frame composition (avoid bottom 15%)
- **On-screen text**: dominant; viewers default to sound-off
- **Cut density**: 1.5–2.5 second average shot length
- **Audio**: loud mix; competes in noisy scrolling environment
- **Captions**: burned-in or platform-overlaid; mandatory
- **Color**: high-contrast; saturated
- **Music**: fast-energy or punctuated silence; not under-score

## Banned Shorts hook patterns

INV-7 + INV-17 apply. Banned:
- "Wait for it"
- "You won't believe"
- "This will change everything"
- "Smash that subscribe" or any subscribe-bait
- "Comment YES if you..." or any engagement-bait
- Fake stakes, manufactured outrage, fabricated shock

Genuine Shorts hook patterns:
- Contrast compressed: "Cheap [X] vs expensive [Y]. Surprising winner."
- Contrarian compressed: "Most [audience] do X. They're wrong."
- Numbered promise: "3 [things] [audience] does that secretly [outcome]."
- Story-beat compressed: "I lost [amount] because of [decision]. Here's the lesson."
- Open question: "Why do [audience] [behavior]? It's not what you think."

## Verification checklist (pre-publish)

- [ ] Hook density: 2+ mechanisms in first 3s + 1 in 3–15s (INV-11 Shorts)
- [ ] No engagement-bait (INV-17)
- [ ] No banned vocabulary (INV-7)
- [ ] On-screen text covers the audio (sound-off viewability)
- [ ] Vertical format confirmed
- [ ] Loop OR bridge designed (not orphan)
- [ ] Title-thumbnail-promise alignment (INV-10) — for Shorts: the cover frame and the audio hook must align
- [ ] Bridge target documented (if not pure-loop)

## Word count

| Length | Target word count |
|---|---|
| 30 seconds | ~75 |
| 45 seconds | ~110 |
| 60 seconds | ~150 |

Shorts run faster wpm than long-form because high-density on-screen text replaces some voiceover.

## Loop completion mechanics

If designed as a loop:

- The closing frame visually links to the opening (color, character, on-screen text)
- The audio's last 1–2 seconds is structurally a setup for the next loop's opening
- The lesson resolves at 80–90% of length; the final 10–20% is loop-set-up
- The loop completion rate (viewers who watch twice) is a Shorts-specific KPI; channels hitting 1.2x+ average loop count perform structurally above the band

## Shorts-to-long-form bridge mechanics

If designed as a bridge:

- Pinned comment with the long-form link (highest-CTR bridge mechanic)
- End-card / final-frame on-screen text directing to long-form
- Description link (visible on tap)
- Channel banner / about-section static direction (for tap-to-channel viewers)

The Short itself is the ad; the long-form is the conversion.

## Shorts as distribution layer

In the YouTube Content OS architecture, Shorts are a distribution layer for installed beliefs — not a conversion layer. The long-form videos do the belief-installation work. Shorts redistribute the installed beliefs to broader audiences who may or may not enter the long-form library.

This means: do not try to install a belief in a Short. Try to teaser a belief that's installed elsewhere. The Short hooks; the long-form converts.

For exception cases (a one-shot lesson that legitimately compresses to 60 seconds), the Short can be self-contained. But the default architecture treats Shorts as bridge content.
