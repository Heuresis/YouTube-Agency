# Pinned VSL Template

> 15-step VSL template. The channel's primary conversion engine. Used for the pinned video that the channel architecture routes most authority videos to. Mode: authority. Target length: 30–60 minutes. Awareness target: solution-aware → product-aware. Goal: convert qualified viewers to applications.

## Frontmatter

```yaml
title: <pinned VSL working title>
mode: authority
length_target_min: 30-60
awareness_target: solution-aware-to-product-aware
structure: <15-step | 11-step | 8-belief>
beliefs_to_install: [Reproduction, Derivative-Desire, Can-Fulfill, Urgency, This-Offer, Better-Faster, Trust-Person, Trust-Expertise]
back_end_offer: <offer name from compartment 3>
application_url: <application form URL>
target_view_to_application_pct: 1-3
```

## Step 1 — Hook (0:00–0:30)

Per INV-11: 3+ retention mechanisms. For pinned VSL, density is higher (5+ mechanisms in first 60s).

```
[VISUAL: <strong opening — the operator on camera, the hook on screen>]

<HOOK — voice-of-viewer pain language; specific outcome promise; 3-brain calibrated>

<BOUND PROMISE — what the viewer leaves with by the end>
```

## Step 2 — Lead (0:30–2:00)

Establish the operator's standing to teach. Why the viewer should listen.

```
<LEAD — 60–90 seconds>

<CREDIBILITY — credentials, lived experience, scale of operation>
<PROOF ANCHOR — 1–2 specific outcomes>
```

## Step 3 — Qualification (2:00–4:00)

Tell the viewer who this is for and who it isn't. Filter early; the wrong audience leaves now.

```
<THIS IS FOR — specific audience description>

<THIS IS NOT FOR — explicit disqualification>

<INVITATION — "if you're [right audience], stay with me; this video will be one of the most useful you watch this year">
```

## Step 4 — Problem & Question (4:00–7:00)

Name the audience's specific problem. Voice-of-viewer language. Activate the loss-aversion / pain.

```
<PROBLEM — 90–120 seconds; voice-of-viewer phrases prominently>

<COST OF INACTION — what's it costing them right now to keep doing what they're doing>

<QUESTION — the structural question the rest of the video answers>
```

## Step 5 — Hero Story & Guru (7:00–12:00)

The operator's origin story or the meta-story of the mechanism's discovery. Calibrated to audience sophistication (low-soph = 5–8 min vulnerability; high-soph = 2–4 min relevant proof).

```
<ORDINARY WORLD — operator's pre-mechanism state>

<DISTURBANCE — what changed; the moment of discovery>

<TRIALS — what didn't work; lived implementation>

<DEATH & REBIRTH — the breakthrough; the mechanism crystallized>

<GIFT — what the operator now has, that the audience can have>
```

## Step 6 — Solution (12:00–17:00)

Introduce the named mechanism / framework. The structural alternative to the niche's standard approach.

```
<SOLUTION INTRODUCTION — name the mechanism>

<HOW IT WORKS — explain at the mechanism level (not application level)>

<WHY IT WORKS — the structural reason — calibrated to the audience's awareness level>
```

## Step 7 — Features & Benefits (17:00–22:00)

What the mechanism does, specifically. Each feature paired with the corresponding benefit (the change in the viewer's life).

```
<FEATURE 1 + BENEFIT 1 — specific, measurable, dated>
<FEATURE 2 + BENEFIT 2>
<FEATURE 3 + BENEFIT 3>
... (3–5 features typical)
```

## Step 8 — Testimonials & Receipts (22:00–28:00)

Social proof. Anonymized cases (segment + scale + niche, never names). Each case matched to a specific objection.

```
<CASE 1 — segment + scale + outcome; addresses Objection X>
<CASE 2 — segment + scale + outcome; addresses Objection Y>
<CASE 3 — segment + scale + outcome; addresses Objection Z>
... (3–5 cases typical; more for $10K+ offers)
```

## Step 9 — Price Anchoring (28:00–34:00)

Per the long-form-VSL specialist: never sell before this step. The price-anchoring mechanic:

```
<ASK FOR ESTIMATE — "what would you guess this would cost?">

<ANALOGIES TO THEIR WORLD — comparing to other investments they make>

<COST OF INACTION COMPARISON — the silent compounding cost of not solving>

<LET THEM CONCLUDE — "you might be thinking $X; that would be reasonable">

<REVEAL PRICE — the actual price; landed as confirmation, not surprise>
```

## Step 10 — Guarantee & Risk Reversal (34:00–37:00)

Reverse the perceived risk. Guarantee structure aligned with offer (typically results-based, satisfaction-based, or extended-trial).

```
<GUARANTEE — specific, time-bound, conditioned>

<RISK REVERSAL — "the only risk is in NOT acting; here's why">
```

## Step 11 — Urgency (37:00–40:00)

Real urgency only — capacity, cohort window, justified price increase. No fake urgency.

```
<URGENCY — structural reason: capacity / cohort / window>

<EXPLANATION — why the urgency is real; the structural constraint>
```

## Step 12 — Crossroads Close (40:00–44:00)

Two paths. Force the decision.

```
<PATH 1 NIGHTMARE — what happens if they do nothing; specific 12-month cost>

<PATH 2 DREAM — what happens if they act; specific 12-month transformation>

<BURDEN OF PROOF — "given these two paths, what would have to be true to NOT act?">
```

## Step 13 — Takeaway Close & Re-Qualification (44:00–47:00)

Disqualify, then re-qualify. The "carrot on a stick."

```
<TAKEAWAY — "this might not be for you. Here's who shouldn't apply">

<RE-QUALIFICATION — "but if you're [specific audience], this is exactly designed for you">
```

## Step 14 — Help Them Buy (47:00–50:00)

Assume the sale. Show how to apply. Remove friction.

```
<APPLICATION INSTRUCTIONS — step-by-step; URL clearly stated>

<PAYMENT OPTIONS — if applicable, shown specifically>

<NEXT STEPS — what happens after they apply>
```

## Step 15 — Future Pace (50:00–55:00)

Walk them through the first 60 seconds after applying, the first week, the first month, the 6-month transformation. Calibrate emotional intensity.

```
<MOMENT 1 — the first 60 seconds after they hit submit>

<WEEK 1 — what they'll experience>

<MONTH 1 — first transformation milestones>

<MONTH 6 — full transformation realized>

<IDENTITY-ASPIRATIONAL CLOSE — who they become>
```

## Step 16 — Final CTA (55:00–end)

Last call. Direct, low-friction.

```
<FINAL CTA — "the application is at [URL]; takes 5 minutes; we're looking for [audience type]">

[ON-SCREEN: URL prominent]
[END-SCREEN: application URL + foundational playlist for those still warming up]
```

## Production cues for pinned VSL

- **Production tier**: highest the operator can sustain (lighting, audio, multi-cam where possible)
- **Visual stability**: continuous set / location; no jarring shifts
- **Energy arc**: rising through Steps 1–8; peak at Steps 9–13; resolution in 14–16
- **Pacing**: slower than reach videos; commits to depth
- **B-roll**: integrated at section transitions; case-study evidence shown visually
- **Music**: under-score throughout; shifts at major belief-installation transitions

## Verification checklist (pre-publish)

- [ ] All 15 steps present and section-tagged
- [ ] Each section's belief-installation mapped to one of the 8 required beliefs
- [ ] Voice-of-viewer language in hook + problem sections
- [ ] No selling before Step 9 (price-anchoring)
- [ ] No fake urgency (INV-5 truth gate)
- [ ] All testimonials anonymized properly (segment + scale + niche; no names)
- [ ] All claims pass 30-second screenshot test (INV-5)
- [ ] No fabrication (INV-6)
- [ ] No banned vocabulary (INV-7)
- [ ] Application URL working
- [ ] Compliance disclaimers as required (finance / health / regulated niches)
- [ ] Predicted AVD ≥ pinned-VSL floor band (typically APV ≥ 40% on 30-60 min)
- [ ] Three-Brain Decision Sequence calibrated in opening

## Length calibration

| Offer tier | Pinned VSL length | Beliefs to fully install |
|---|---|---|
| $1K–$3K | 25–35 minutes | 6–8 |
| $3K–$10K | 35–50 minutes | 8 (all) |
| $10K–$30K | 45–60 minutes | 8 + niche-specific reinforcement |
| $30K+ | 60+ minutes; consider multi-VSL series instead | 8 + custom belief stack |
